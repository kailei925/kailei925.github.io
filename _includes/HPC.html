TROUBLESHOOTING RECOMMENDATIONS

APPENDIX: QUICK TROUBLESHOOTING GUIDE
General steps. Listed below are a basic order of steps to follow when attempting to stabilize a simulation. Re-run the simulation with the recommended changes at each step. If still unstable, move to the next step, reum, and repeat as needed:

  1. Review input wave, water level, and spatial resolution conditions per Torres et al. (2022)
  1. Reduce CFL to 0.25 and FroudeCap to 1.5
  1. Reduce CFL to 0.1
  1. Us built-in localized topobathy smoothing
    a. BATHY CORRECTION = T
    b. SmoothBelowDepth = - 1,0
    c. SlopeCap = 1.0
  1. Manually smooth data using some global technique, for example, Gaussian filter/smoother (see FUNTAVE_TVD/tools/filter in GitHub repository)
  1. Manually adjust topobathy data in the problem region
  1. Submit you problem to the FUNWAVE User Group listserv. Signup is available on the Wiki.

Visualzing problem area. If identifying the instability from visualizing the *_99999 files is challenging follow the below steps:

  1. Get instability time from LOG.txt (Figure 2. BlowUp Time).
  1. Set PLOT_START_TIME to just before instability time. Use last valid outputs as a guide.
  1. Roduce PLOT_INTY, for example, 10-30 plots between PLOT_START_TIME and instability time. Visualize output.
  1. Repeat if necessary further adjusting PLOT_START_TIME and PLOT_ INTV.

TROUBLE SHOOTING

MaxEta and MinEta

  • If MinEla values steadily decrease below the input wave height or reach the water depth, this may indicate or be a precursor to an instability Max U and Max V
  • Unrealistic (eg., very large) values for Max U and Max V can indicate a fundamental error in the setup of the simulation
  • Large values may not always trigger a failure mode
    Froude
    â€¢ If Froude â‰¥ FroudeCap (1.0 - 3.0) indicates instability

Mass Volume

  • If water appears to be draining from the domain over the course of the simulation, there is likely an imbalance in the mass conservation formulation related to wetting/drying.
  • Change in MassVolume >10% is considered significant
  • Gaps between bathymetry/topography and boundary edges may be a source of imbalance or instability

POTENTIAL SOURCES OF INSTABILITY
Bathymetry

  • Rapid changes in bathymetry and steep slopes are challenging for Boussinesa models
  • Apply localized or global smoothing to reduce or eliminate "jumps" in bathy
  • If instability still occurs after global smoothing
  • Consider manual modification of the bathymetry, or
  • Revisit input bathymetry to identify potential issues
  • e.g, incorrectly merged bathymetry sources, leading to stair-stepping of values

Wetting/Drying

  • Model initializes with a thin layer of fluid everywhere in the domain, including areas above the still water line
  • Model numerically dries grid points that are below a specified depth threshold of thickness (MinDepth)
  • Sections of emergent topography experiencing intermittent or limited wetting events are susceptible to either BlowUp Time or NaN in Statistics failure mode
  • Ex: "dry" bucket behind the levee experienced intermittent inundation due to overtopping, causing a seich in the bucket that accelerated a MaxAbsEta failure mode.
  • Increasing surface friction in problem areas may help dampen wave energy and prevent instability.

Potential solutions:

  • Adjust thickness of fluid (MinDepth)
  • Recommended minimum: 0.01
  • Increasing MinDepth decreases sensitivity of the wetting/drying condition
  • If wave runup is of interest, reducing the MinDepth yields a more accurate solution but may be less stable, particularly on steep slopes.
  • Adjust elevation (topobathy) in problem area
  • Adjust initial water level elevation

GENERAL RECOMMENDATIONS
Station files

  • ASCI text file where each row specifies the location of a station in FUNWAVE (i,j) coordinates
  • NEW Station OutputBuffer parameter stores data in memory until buffer limit is reached, then prints to stations output
    file (sta_ xxxx)
  • Default: StationOutputBuffer = 1000 (int)
  • Extra zeroes can be printed in the station files if the runtime is less than the buffer time
  • Dt is dynamic, not constant, between outputs

Use periodic boundary conditions in 2D where appropriate

  • This allows for waves to propagate throughout the whole domain once a sea state develops
  • If using periodic boundary conditions, the domain and bathymetry must also be periodic.
  • "Wrapping" bathymetry boundaries creates smooth variation along the lateral boundaries so that the north lateral boundary is continuous with the south lateral boundary.
  • Sharp changes in bathymetry must be smoothed.
  • If not using periodic boundary conditions, be sure to use sponge layers.
  • This is not always recommended because some energy is lost to the sponges.

Don't let your sponge layer overlap with the wavemaker location!

Advise H/h < 0.5
â€¢ If you exceed this value, generated waves can break within the wavemaker.

Bathymetry, friction, breakwater, and obstacle files all must be of same dimensions (mglob x nglob)

For bathymetry, check that slopes are below Ï€/2.
